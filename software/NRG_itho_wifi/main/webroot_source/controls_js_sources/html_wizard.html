RF Remote Setup

RF remotes are wireless remotes emulated by the add-on using the CC1101 RF module. By pairing the add-on as an RF remote and setting the "Remote function" to "Send", it can send commands (speed, timer) wirelessly to your Itho unit.

Setting the "Remote function" to "Receive" ensures commands from a physical remote will be received, processed, and translated to the proper commands for your Itho unit by the add-on. Make sure the remote is unpaired from your Itho to prevent command conflicts.

If you set the "Remote function" to "Monitor only", commands are received by the add-on and relevant information is published on the APIs and web interface. No action is taken by the add-on to control the Itho; leave the remote paired with the Itho unit in that case.

You can pair RF remotes here, or skip this step and do it later via the RF Devices menu.

MQTT Setup

MQTT is a lightweight messaging protocol. The add-on uses MQTT to publish device status (fan speed, temperatures, errors) and receive commands (speed changes, timer activation).

If you use Home Assistant, Domoticz, or another home automation platform, install a MQTT server, often available as plugin for your home automation platform, and activate the server to integrate your Itho device. The "Server" input field should contain the IP address of your MQTT server. This is often the IP address of your Home Assistant, Domoticz etc. device. The Homey app store offers a plugin that uses the WebAPI, for that setup MQTT is optional.
